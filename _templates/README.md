# 📝 筆記範本使用指南

這個資料夾包含了各種筆記範本和快速創建工具，幫助你更有效率地撰寫部落格文章和工作日誌。

---

## 📂 範本類型

### 1. 文章範本

| 範本檔案 | 用途 | 適用場景 |
|---------|------|---------|
| `post-general.md` | 通用筆記 | 一般性的思考、筆記、想法記錄 |
| `post-reading.md` | 閱讀心得 | 書籍、文章的閱讀筆記和心得 |
| `post-tcm.md` | 中醫學習 | 中醫理論、臨床觀察、案例分析 |
| `post-ai.md` | AI 探索 | AI 技術實驗、應用、研究筆記 |
| `post-life.md` | 生活觀察 | 日常觀察、生活思考、洞察記錄 |
| `micro-post.md` | 短訊息 | Plurk 風格的短文、隨筆（280 字內）✨ |

### 2. 工作日誌範本

| 範本檔案 | 用途 |
|---------|------|
| `workjournal.md` | 每日工作記錄、進度追蹤、心得總結 |

---

## 🚀 快速開始

### 方法一：使用自動化腳本（推薦）

#### 創建新文章
```bash
# 在專案根目錄執行
bash _templates/new-post.sh
```

**步驟**：
1. 選擇文章類型（1-5）
2. 輸入文章標題（英文，用於檔名）
3. 自動創建並填入今天的日期
4. 開始編輯內容！

#### 創建工作日誌
```bash
# 在專案根目錄執行
bash _templates/new-journal.sh
```

**步驟**：
1. 選擇日期（今天或自訂）
2. 自動創建日誌檔案
3. 開始記錄工作內容！

#### 💬 快速發布短訊息（新功能！）
```bash
# 在專案根目錄執行
bash _templates/new-micro.sh
```

**特色**：
- 類似 Plurk 的發文體驗
- 單行或多行輸入模式
- 自動時間戳記
- 字數統計（建議 280 字內）
- 自訂標籤
- 即時預覽

---

### 方法二：手動複製範本

#### 創建文章
```bash

```

#### 創建工作日誌
```bash
# 1. 複製範本到 _workjournal 目錄
cp _templates/workjournal.md _workjournal/2026-02-08.md

# 2. 編輯檔案，填寫工作內容
```

## 📋 範本結構說明

### 文章範本共同元素

所有文章範本都包含以下基本結構：

```markdown
---
title: "文章標題"
date: YYYY-MM-DD
author: Frank Lee
category: 分類名稱
tags: [標籤1, 標籤2]
layout: post
---

# 標題

內容區塊...

---

**記錄日期**: YYYY-MM-DD
```

### 特殊範本說明

#### 📚 閱讀筆記 (`post-reading.md`)
- 包含書籍資訊、摘錄、反思
- 適合深度閱讀記錄
- 有推薦指數評分

#### 🏥 中醫學習 (`post-tcm.md`)
- 包含理論基礎、病機分析
- 臨床觀察、治療思路
- 案例分析（選填）

#### 🤖 AI 探索 (`post-ai.md`)
- 包含技術概述、實驗過程
- 程式碼片段、結果分析
- 應用場景探討

#### 🌱 生活觀察 (`post-life.md`)
- 輕鬆的記錄格式
- 強調觀察與思考的連結
- 可加入行動計畫

#### 💬 短訊息 (`micro-post.md`) ✨新增
- 極簡格式，快速發布
- 建議 280 字以內
- 自動時間戳記
- 適合隨筆、突發想法、快速記錄

## 💬 短訊息使用範例

### 情境一：快速記錄想法

```bash
$ bash _templates/new-micro.sh

請選擇輸入方式：
1) 直接輸入短文（建議 280 字以內）

請選擇 (1-2): 1

✓ 內容長度：89 字

✅ 短訊息已發布！
```

```bash

剛才在診所遇到一個案例
患者主訴疲勞、腹脹
舌淡苔白，脈緩弱
典型的脾氣虛證
END

✅ 短訊息已發布！
```

## 🎨 自訂範本

你可以根據需要創建自己的範本：

1. 在 `_templates` 目錄創建新的 `.md` 檔案
2. 參考現有範本的結構
3. 加入你需要的區塊和格式
4. 儲存後即可使用

## 💡 使用技巧

### 1. 檔名規範
- **文章格式**：`YYYY-MM-DD-title-slug.md`
- **短訊息格式**：`YYYY-MM-DD-micro-YYYYMMDD-HHMMSS.md`（自動生成）
- 日期：使用文章撰寫日期
- 標題：使用英文小寫，單字間用 `-` 連接
- 範例：`2026-02-08-my-first-post.md`

### 2. Front Matter 設定
```yaml
---
title: "文章標題"        # 顯示在頁面上的標題
date: 2026-02-08        # 文章日期
author: Frank Lee       # 作者名稱
category: 分類名稱      # 單一分類
tags: [標籤1, 標籤2]    # 多個標籤
layout: post            # 使用的版型
---
```

### 3. 圖片使用
```markdown
# 插入圖片
![圖片說明](/frank-lee-notes/assets/images/圖片檔名.jpg)

# 圖片存放位置
assets/images/圖片檔名.jpg
```

# 設定工作日誌腳本權限
chmod +x _templates/new-journal.sh

# 設定短訊息腳本權限
chmod +x _templates/new-micro.sh

## 🎯 最佳實踐

1. **保持一致性**：使用範本確保文章結構一致
2. **及時記錄**：想到就記，不要拖延
3. **適度調整**：範本可以根據需要增刪區塊
4. **善用標籤**：方便日後搜尋和分類
5. **定期回顧**：工作日誌可以幫助追蹤進度
6. **短訊息精簡**：保持在 280 字以內，聚焦核心想法 ✨
